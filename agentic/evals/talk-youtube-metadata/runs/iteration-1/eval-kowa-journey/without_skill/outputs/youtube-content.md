## Description

What does it actually look like when a working software team adopts AI coding tools day to day? In this JB Agentic meetup talk, Kowa Jia Liang, a senior software engineer at Payboy (a Singapore-based HR SaaS company), walks through his team's real evolution from manual coding to ChatGPT, Cursor, and finally Claude Code.

Kowa shares the concrete setup his team uses to ship features faster on a Ruby on Rails + Vue.js codebase, including the trade-offs they hit along the way and why they ultimately chose quality over raw speed.

In this talk:
- The evolution of AI in their workspace: manual coding → ChatGPT (context-blind copy-paste) → Cursor IDE → VS Code + Claude Code
- Permanent context with CLAUDE.md: storing tech stack, architecture, portals, and code review checklists so the agent already knows what's expected
- Custom skills organized by module (attendance, claims, etc.) to store product logic and trigger on demand
- Case study: a "changelog" skill that auto-summarizes PR changes
- Case study: a "code-auditor" skill for pre-PR review that eliminates 70-80% of redundant review comments
- Using the Superpower plugin to plan and build a delayed MRR reporting feature in about 30 minutes, tests included
- MCP tools explained (the "USB-C for AI apps") and a plan to connect Sentry for faster bug triage
- Why they moved from Cursor to Claude Code: sub-agents and context management vs. speed and autocomplete
- Honest Q&A on token costs, CLAUDE.md file size and the 200-line guideline, on-demand references, QA/staging before production, and testing Codex as a cheaper alternative

This is a practical, no-hype look at how AI coding tools fit into a real product team's workflow, with candid notes on what works, what still breaks, and what needs ongoing maintenance.
