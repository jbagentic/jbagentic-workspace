## Description (copy-paste ready)

```
A senior engineer at an HRMS SaaS company walks through his team's real AI coding journey — from manual coding and copy-pasting into ChatGPT, to Cursor, to VS Code with Claude Code — and the setup that finally made AI useful in production.

Kowa Jia Liang (Payboy, Singapore) shows the CLAUDE.md permanent-context file, custom Claude skills (a changelog generator and a code-auditor), the Superpowers plan-then-build workflow, and wiring Sentry in through MCP — with an honest Cursor-vs-Claude comparison and a long, practical Q&A.

Talk: "From Manual Coding to ChatGPT, Cursor and Claude Code" by Kowa Jia Liang

What's covered:
- The evolution: manual coding → ChatGPT → Cursor → Claude Code
- CLAUDE.md as a permanent context file (and the struggle to keep it near 200 lines)
- Custom skills per module, plus a code-auditor that wipes out redundant PR comments
- Superpowers: a plan you review first, then agents build it with test cases
- An MRR report page shipped in ~30 minutes after being stalled for months
- Connecting Sentry through MCP to debug straight from production logs
- Why a product-heavy team picked Claude (quality) over Cursor (speed)
- Q&A: CLAUDE.md contents, splitting skills by module, Codex, token cost, maintenance
```
