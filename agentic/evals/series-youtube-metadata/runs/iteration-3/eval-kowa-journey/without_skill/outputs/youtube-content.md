A senior engineer at an HRMS SaaS company walks through his team's real AI coding journey — from manual coding and pasting snippets into ChatGPT, to Cursor, to VS Code + Claude Code — and the setup that actually made it stick in production.

🎤 Talk: "Our AI Journey: From Manual Coding to ChatGPT, Cursor, and Claude" by Kowa Jia Liang

Kowa Jia Liang (Payboy, a Ruby on Rails + Vue HRMS product in Singapore) shows the CLAUDE.md permanent context file, custom Claude skills split per module, a code-auditor that cuts 70–80% of redundant PR comments, the Superpowers plan-then-build workflow, and connecting Sentry over MCP to debug from production logs — plus an honest Cursor-vs-Claude comparison and a long, practical Q&A.

What's covered:
- The evolution: manual coding → ChatGPT → Cursor → VS Code + Claude Code
- Browser-era AI vs editor-native AI: why context that lives in your repo wins
- CLAUDE.md as a permanent context file — and the case for keeping it under 200 lines with on-demand references
- Custom skills per module, each holding the product logic and trigger keywords
- A code-auditor skill: a pre-PR review that wipes out 70–80% of redundant comments
- Superpowers: a plan you sign off on, then parallel agents build, test, and fix
- A stalled MRR report page shipped in ~30 minutes
- Connecting Sentry through MCP to debug straight from production logs
- Why a product-heavy team chose Claude (quality) over Cursor (speed)
- Q&A: CLAUDE.md content, splitting skills, Superpowers tokens, Codex pricing, staging/QA, Graphify caching
