## Description (copy-paste ready)

How one HRMS SaaS team went from copy-pasting into ChatGPT to running Claude Code across a real production codebase. Kowa Jia Liang walks through the exact setup his team uses every day — a CLAUDE.md permanent-context file, custom per-module skills, a pre-PR review skill, the Superpower plan-then-build workflow, and MCP tools wired into Sentry — and why they chose quality over speed.

🎤 Talk: "Our AI Journey: From Manual Coding to ChatGPT, Cursor, and Claude" by Kowa Jia Liang
📍 JB Agentic Meetup #1 — 30 May 2026

What's covered:
• The evolution: manual coding → ChatGPT → Cursor → VS Code + Claude Code
• Why editor-native AI beats browser chat: it already knows your codebase
• CLAUDE.md as permanent context — and keeping it under 200 lines with references
• Custom skills per module so the agent retrieves the right product logic
• A changelog skill and a code-auditor skill that kills 70–80% of redundant PR comments
• Superpower: digest → plan → sign off → parallel agents build it (an MRR report page in ~30 min)
• MCP tools and a Sentry integration for faster, guess-free bug fixing
• Cursor vs Claude: sub-agents, context, and choosing quality over speed

Tech stack: Ruby on Rails, Vue.js, Claude Code, Cursor, MCP, Sentry, Superpower
