## Description

Terraform is powerful but writing IaC by hand means reading docs, clicking portals, and juggling tickets. What if an AI agent could read your requirements from a markdown file and generate the Terraform code for you? Khew Jia Peng demos exactly that — a proof-of-concept where a Codex agent reads structured requirements, writes Terraform, validates with `terraform plan`, and even updates the documentation automatically.

🎤 Talk: "AI Meets Infrastructure — Integrating IaC, AI Agents, and Markdown" by Khew Jia Peng
📍 JB Agentic Meetup #1 — 30 May 2026

What's covered:
• What Infrastructure as Code (IaC) is and why it beats portal clicking
• Writing infrastructure requirements in markdown for AI agents to consume
• Live demo: Codex agent reads a new requirement and generates Terraform code
• Agent behaviour rules and skills markdown to keep production safe
• Terraform plan/validate/apply workflow with PR review before deploy
• CI/CD pipeline as the next step to close the loop
• Traditional manual provisioning vs AI-assisted infrastructure workflow
