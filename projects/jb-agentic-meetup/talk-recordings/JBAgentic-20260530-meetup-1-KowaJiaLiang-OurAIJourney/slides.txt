## slide-14
❖
Whenever our team trying to do a new feature, normally we will use this `
superpower plugin
`.
❖
`
Superpower
` is a software development methodology for coding agent, its a public repo on github
❖
How it work:
➢
When you trigger this superpower, first he will digest your requirement (from chat), then they will
give you a proposal, or we can say its a coding plan with details like what files he will be creating
(according to your coding standard rules), come up with core logic functions with test cases in a
.md file.
➢
After you reviewed the .md file, you can chat and make any changes you want, then you sign off
(to give green light to agent & proceed).
➢
Finally, Claude Agent will follow the .md design file to start coding & designing, then Claude will
create parallel agents to test the written code, fix bug & test & repeat until all good.

## slide-20
Why We Moved: Cursor vs. Claude
Metric
Cursor (The Typist IDE)
Claude (The System Architect)
Active Chat Context
blocks, leading to slow replies. Whenever you
doing a big feature, the response will not be too
accurate
Context Isolation: Spawns parallel background
subagents to work behind the scenes & give
answer only to the origin chat agent.
xecution Logic
Inline focus: Predicts autocomplete steps
dynamically while typing. (Very good for inline
logic & redundant codes)
Slower, no auto-complete suggestion
for small prompt sizes, quick debugging, and
micro-queries.
Slower but Smarter: Takes longer to process, but
the resulting logic is significantly more accurate.
Understand that Cursor & Claude actually behave quite similar, but in the end we still choosing claude for our
the end we choose quality over speed.
