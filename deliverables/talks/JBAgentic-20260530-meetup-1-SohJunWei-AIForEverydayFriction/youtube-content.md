## Description (copy-paste ready)

```
Build a personal expense tracker you operate by texting, voicing, or photographing a receipt to a Telegram bot — no app to open, no manual entry. Soh Jun Wei walks through the real build: a Hermes AI agent that reads fuzzy input, and an MCP server that does the deterministic work (validating categories, doing the math, writing to the database).

🎤 Talk: "AI for Everyday Friction" by Soh Jun Wei
📍 JB Agentic Meetup #1 — 30 May 2026

What's covered:
• Killing friction: turn "lunch 5.5" (text, voice, or a receipt photo) into a saved expense
• Architecture: Telegram → Hermes agent → MCP → SQLite → Next.js dashboard, hosted free on Oracle Cloud
• Why prompting alone breaks, and why MCP gives the agent predictable tool boundaries
• The core split: AI for fuzzy input (language, voice, receipts), tools for deterministic actions (validation, math, DB writes)
• MCP vs CLI — when each makes sense
• Speech-to-text options: local Whisper vs Groq's free Whisper-large API
• From coder to orchestrator: how the developer's job shifts to designing how AI is allowed to act
```
