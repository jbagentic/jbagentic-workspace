## Description (copy-paste ready)

```
A senior engineer at an HRMS SaaS company walks through his team's real AI coding journey — from manual coding and copy-pasting into ChatGPT, to Cursor, to VS Code + Claude Code — and the setup that actually made it work in production.

Kowa Jia Liang (Payboy, Singapore) shows the CLAUDE.md permanent context file, custom Claude skills (changelog, code-auditor), the Superpowers plan-then-build workflow, and connecting Sentry via MCP — plus an honest Cursor-vs-Claude comparison and a long, practical Q&A.

Talk: "From Manual Coding to ChatGPT, Cursor and Claude Code" by Kowa Jia Liang

What's covered:
- The evolution: manual coding → ChatGPT → Cursor → Claude Code
- CLAUDE.md as a permanent context file (and keeping it under 200 lines)
- Custom skills per module, and a code-auditor that cuts 70–80% of redundant PR comments
- Superpowers: a plan you sign off on, then parallel agents build and test it
- An MRR report page shipped in ~30 minutes after being stalled for months
- Connecting Sentry through MCP to debug from production logs
- Why a product-heavy team chose Claude (quality) over Cursor (speed)
```
